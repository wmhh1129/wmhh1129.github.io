如何舒适的用git进行项目管理

本文从日常开发的角度，阐述平时日常工作流程中的常用git操作(阐述rebase流派的：）

  • 下面的操作涉及了git工具（可视化跟命令行both ok） & github 账号

  • 首先假定工作组有2-3人，且你们都有github的账号，以及项目repo的代码提交权限.

远程项目的master分支：（代码打包或部署上线的最后一道关，一般不涉及上线一般不会去动它）
远程项目的develop分支：（这就是我们每天打交道的开发分支）

  • 现在我们有开发需求啦，需要切分支进行不同feature特性的开发

首先克隆一份远程仓库的代码到本地，以后就基本不需要进行这一步的操作。
git clone <repository address>
然后 切入我们的开发分支,（开发新特性记得基于最新的develop哦）.
git checkout develop
为了让本地的develop分支是跟远程保持一致的，可以进行
git pull （pull = fetch + merge）
重点来了，这个操作是基于最新develop切出一个属于你自己的独立特性分支, 当然这个分支名只是我举的例子而已
git checkout -b feature/add-remark

  • 好的，相信你已经在你独立的特性分支上大展拳脚，做出了酷炫的功能。

那么这个时候你的工作区肯定跟刚拉取下来的代码大相径庭，先看看有哪些不同。
git status
如果有红色modified的地方，根据需要用添加在暂存区
git add <filename> 或 git add .(全部加入暂存区)
此时可以继续用git status命令查看状态，如果想要添加的文件都变绿色了，则需要下一步操作
git commit -m 'feat: add remark function' (示例都比较贴近分支与commit规范)

  • 现在已经形成了一个commit，要将这个改动推送在远程仓库

    git push origin feature/add-remark(约定俗成远程与本地分支名保持一致,origin后面为新建远程分支名)
